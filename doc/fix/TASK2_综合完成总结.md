# TASK 2 - 选股历史功能综合完成总结

## 任务概述
修复选股历史页面中的多个问题，包括选入价格、最高最低价、实时价格和行业板块信息的显示。

## 问题清单与解决方案

### 问题 1：选入价格显示为 ¥0.00
**原因**：前端显示的是 `selection_price` 字段，该字段在数据库中为 0

**解决方案**：
- 后端计算 `selection_day_price`（选入当日收盘价）
- 前端优先使用该字段显示选入价格
- 从CSV数据中获取选入当日的收盘价

**状态**：✅ 已完成

---

### 问题 2：最高价和最低价计算错误
**原因**：原始代码使用 `>` 比较，导致选入当日的数据被排除

**解决方案**：
- 修改计算逻辑：从选入日期**之后**到今天（包含选入当日）
- 当无历史数据时，使用腾讯财经实时查询
- 基于选入当日收盘价计算收益率

**状态**：✅ 已完成

---

### 问题 3：当前价格显示不是实时价格
**原因**：缺少 `get_stock_price()` 方法，无法从腾讯财经获取实时价格

**解决方案**：
- 添加 `AKShareFetcher.get_stock_price()` 方法
- 使用腾讯财经接口获取实时价格
- 根据交易时间自动切换实时/历史价格

**状态**：✅ 已完成

---

### 问题 4：行业板块信息显示为"- / -"
**原因**：AKShare API不稳定，缺少从腾讯财经获取的机制

**解决方案**：
- 添加 `AKShareFetcher.get_stock_industry_sector()` 方法
- 使用腾讯财经接口获取股票的行业和板块信息
- 实现多级缓存和降级策略

**状态**：✅ 已完成

---

## 实现细节

### 后端修改

#### 1. `utils/akshare_fetcher.py`
**新增方法**：
- `get_stock_price(stock_code: str) -> float`
  - 使用腾讯财经接口获取实时价格
  - 支持沪深两市股票代码
  - 返回实时价格，获取失败返回 None

- `get_stock_industry_sector(stock_code: str) -> Tuple[str, str]`
  - 使用腾讯财经接口获取行业和板块信息
  - 支持沪深两市股票代码
  - 返回 (industry, sector) 元组

#### 2. `utils/selection_record_manager.py`
**改进方法**：
- `calculate_performance()`
  - 添加 `selection_day_price` 字段（选入当日收盘价）
  - 修改高低价计算：从选入日期之后到今天
  - 使用实时价格计算收益率

- `_get_current_price()`
  - 根据交易时间判断获取实时价格或历史价格
  - 交易时间（9:30-15:00）：获取实时价格
  - 收盘后（15:00+）：使用当日收盘价
  - 开盘前（<9:30）：使用前一日收盘价

- `_get_stock_info()`
  - 优先从本地缓存获取
  - 缓存不存在时，从腾讯财经实时获取
  - 腾讯财经失败时，降级到AKShare获取
  - 获取成功后，更新本地缓存

### 前端修改

#### `web/static/js/selection_history.js`
- 修改 `renderTable()` 方法
  - 使用 `selection_day_price` 作为选入价格显示
  - 使用 `current_price` 作为当前价格显示
  - 移除收益率显示（暂时不显示）

#### `web/templates/selection_history.html`
- 增加 JS 版本号强制刷新浏览器缓存

### API修改

#### `web_server.py`
- 改进 `get_selection_history()` 端点
  - 添加 numpy 类型转换，确保 JSON 序列化正确

## 腾讯财经接口说明

### 实时价格接口
```
URL: https://qt.gtimg.cn/q={query_code}
查询代码: sh{stock_code} (上海) 或 sz{stock_code} (深圳)
返回格式: v_sh600519="~...~当前价~..."
```

### 行业板块接口
```
URL: https://qt.gtimg.cn/q={query_code}
返回格式: v_sh600519="~...~行业~地域~概念~..."
字段说明:
  - parts[3] = 当前价格
  - parts[20] = 行业
  - parts[21] = 地域
  - parts[22] = 概念
```

## 已知限制

1. **腾讯财经API限制**
   - 可能存在请求频率限制
   - 网络不稳定时可能失败

2. **行业板块信息延迟更新**
   - 只有在查询历史选股时才会更新
   - 不支持主动刷新所有股票的信息

3. **实时价格延迟**
   - 交易时间外显示前一日收盘价
   - 可能存在1-2秒的延迟
