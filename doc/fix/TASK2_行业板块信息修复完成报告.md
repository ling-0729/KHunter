# TASK 2: 选股历史行业/板块信息修复 - 完成报告

## 任务概述

修复选股历史页面中行业/板块信息显示为"- / -"的问题，实现从权威数据源（AKShare）实时获取行业/板块信息。

## 问题分析

### 根本原因
- 原有的 `industry_sector_map.json` 硬编码映射表存在严重错误
- 所有股票都被错误地分类为"纺织/制造"
- 数据库中存储的行业/板块信息不准确

### 具体表现
- 前端显示"- / -"（空值）
- 数据库中有619条记录的行业/板块信息为空或错误
- 示例：金岭矿业(000655)、惠天热电(000692)被错误分类为"纺织/制造"

## 解决方案

### 核心设计原则
1. **权威数据源**：使用AKShare作为行业/板块信息的唯一权威来源
2. **按需获取**：不进行批量更新，只在查询历史选股时实时获取缺失的信息
3. **本地缓存**：将获取的信息缓存到 `stock_names.json`，避免重复API调用
4. **容错处理**：AKShare连接失败时优雅降级，不影响系统正常运行

### 实现细节

#### 1. 修改 `get_selection_history()` 方法
- 查询历史选股时，检查每条记录的行业/板块信息
- 如果缺失，调用 `_get_stock_info()` 实时获取
- 获取成功后，同时更新数据库和缓存

```python
# 如果缺少行业/板块信息，实时获取
if not record.get('industry') or not record.get('sector'):
    industry, sector = self._get_stock_info(record['stock_code'])
    if industry or sector:
        record['industry'] = industry
        record['sector'] = sector
        # 同时更新数据库
        self._update_stock_in_db(record['stock_code'], industry, sector)
```

#### 2. 新增 `_update_stock_in_db()` 方法
- 更新数据库中该股票的行业/板块信息
- 使用 `updated_at` 字段记录更新时间

```python
def _update_stock_in_db(self, stock_code: str, industry: str, sector: str):
    """更新数据库中该股票的行业/板块信息"""
    update_sql = """
    UPDATE stock_selection_record 
    SET industry = ?, sector = ?, updated_at = ?
    WHERE stock_code = ? AND is_active = 1
    """
    cursor.execute(update_sql, (industry, sector, datetime.now(), stock_code))
    self.conn.commit()
```

#### 3. 数据清理
- 清除数据库中所有619条记录的错误行业/板块信息
- 保留 `stock_names.json` 中的股票代码和名称，清空行业/板块字段

## 工作流程

### 用户查询选股历史时的流程
1. 前端请求选股历史数据
2. 后端查询数据库，获取历史记录
3. 对每条记录检查行业/板块信息
4. 如果缺失：
   - 从 `stock_names.json` 缓存查找
   - 如果缓存中没有，从AKShare获取
   - 获取成功后，更新数据库和缓存
5. 返回完整的历史记录给前端

### 缓存策略
- **一级缓存**：`stock_names.json` 本地文件
- **二级缓存**：数据库中的 `industry` 和 `sector` 字段
- **实时获取**：AKShare API（仅在缓存缺失时）

## 性能考虑

### 优化措施
1. **按需获取**：避免批量更新所有5161个股票
2. **本地缓存**：减少API调用次数
3. **数据库缓存**：查询历史时直接从数据库读取已缓存的信息
4. **容错处理**：API失败不影响系统运行

### 预期性能
- 首次查询某股票历史：可能需要调用AKShare API（1-2秒）
- 后续查询同一股票：直接从数据库读取（<100ms）

## 已知限制

1. **AKShare API不稳定**
   - 连接经常超时或断开
   - 建议在网络良好的环境下使用

2. **行业/板块信息延迟更新**
   - 只有在查询历史选股时才会更新
   - 不支持主动刷新所有股票的信息

3. **无法处理已删除的股票**
   - 如果股票从市场退市，AKShare可能无法获取信息
