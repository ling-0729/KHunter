# 上升趋势判断算法 Bug 报告

**报告日期**: 2026-03-21  
**问题**: 上升趋势判断逻辑有误，导致明显下跌的股票被判定为上升趋势  
**影响**: 15 只被判定为上升趋势的股票中，至少有 2 只（000049、000498）不符合上升趋势

---

## 问题描述

### 案例1: 000498 股票
- **20天涨幅**: **-3.44%** (明显下跌)
- **被判定为**: 上升趋势 ✅ (错误！)
- **最新价**: 5.90 (下跌)

### 案例2: 000049 股票
- **20天涨幅**: +5.03% (整体上升)
- **但最近趋势**: 下跌 (-3.32%, -0.15%)
- **被判定为**: 上升趋势 ✅ (可能错误)

---

## 根本原因分析

### 问题所在

`_detect_wave_points()` 方法的逻辑有缺陷：

```python
def _detect_wave_points(self, prices):
    # 将数据反转为正序（从早到晚）
    prices_asc = prices.iloc[::-1].reset_index(drop=True)
    
    # 遍历数据，识别波峰和波谷
    for i in range(1, len(prices_asc) - 1):
        current = prices_asc.iloc[i]
        prev = prices_asc.iloc[i - 1]
        next_val = prices_asc.iloc[i + 1]
        
        # 波峰判断：当前 > 前 AND 当前 > 后
        if current > prev and current > next_val:
            wave_points.append(('peak', i, float(current)))
```

### 问题分析

1. **数据反转问题**: 
   - 输入数据是从新到旧排列（最新在前）
   - 反转后变成从旧到新排列
   - 但波峰波谷的识别逻辑可能不正确

2. **波峰波谷识别不准确**:
   - 只看相邻的三个点（前、当前、后）
   - 无法识别真正的趋势
   - 可能识别出虚假的波峰波谷

3. **上升趋势判断不严格**:
   - 即使波峰波谷递增，也不能说明是上升趋势
   - 需要检查整体的价格方向

---

## 解决方案

### 方案 A: 改进波峰波谷识别（推荐）

使用更严格的标准：
1. 波峰：当前价格 > 前3日最高价 AND 当前价格 > 后3日最高价
2. 波谷：当前价格 < 前3日最低价 AND 当前价格 < 后3日最低价

### 方案 B: 添加整体趋势检查

在波峰波谷递增的基础上，还要检查：
1. 最新价格 > 20日平均价格
2. 最新价格 > 20日最低价格的 50% 以上

### 方案 C: 简化上升趋势判断

直接检查：
1. 最新收盘价 > 20日平均收盘价
2. 最新收盘价 > 20日最低价格
3. 最近5日平均价格 > 前5日平均价格
