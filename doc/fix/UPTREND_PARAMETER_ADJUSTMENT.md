# 上升趋势判断参数调整总结

**调整时间**: 2026-03-21  
**调整原因**: 排除虚假信号（000020, 000062, 000626等反弹股票）  
**调整状态**: ✅ 完成  

---

## 问题分析

### 用户反馈
- 000020, 000062, 000626都不属于上升趋势
- 这些股票虽然有反弹，但前面有明显下跌
- 希望参数更严格

### 根本原因
旧参数过于宽松，将"反弹"误判为"上升趋势"：
- 000020: 20日收益率+55.53%，但最大回撤9.93%
- 000062: 20日收益率+5.80%，最大回撤8.89%
- 000626: 20日收益率+6.58%，最大回撤9.57%

这些都是"先跌后涨"的反弹，不是真正的上升趋势。

---

## 参数调整

### 旧参数（宽松）
```yaml
uptrend_pvalue_threshold: 0.05        # p值 < 0.05
uptrend_rsquared_threshold: 0.3       # R² > 0.3
# 无收益率限制
# 无回撤限制
```

### 新参数（严格）
```yaml
uptrend_pvalue_threshold: 0.01        # p值 < 0.01 ✅ 更严格
uptrend_rsquared_threshold: 0.5       # R² > 0.5 ✅ 更严格
uptrend_min_return_threshold: 0.08    # 最小收益率 > 8% ✅ 新增
uptrend_max_drawdown_threshold: 0.08  # 最大回撤 < 8% ✅ 新增
```

### 调整说明

| 参数 | 旧值 | 新值 | 说明 |
|------|------|------|------|
| p值阈值 | 0.05 | 0.01 | 要求趋势更显著 |
| R²阈值 | 0.3 | 0.5 | 要求拟合度更好 |
| 最小收益率 | 无 | 8% | 排除低收益反弹 |
| 最大回撤 | 无 | 8% | 排除大幅回撤 |
