# 日志系统使用说明

## 概述

后端已实现完整的日志系统，支持文件持久化、日志轮转和多级别日志记录。

## 日志位置

### 后端日志
- **文件位置**: `logs/app.log`
- **日志级别**: DEBUG、INFO、WARNING、ERROR
- **轮转策略**: 单个文件最大 10MB，保留最近 5 个备份文件
- **格式**: `时间 - 模块 - 级别 - [文件:行号] - 消息`

### 前端日志
- **位置**: 浏览器开发者工具（F12 → Console 标签）
- **包含内容**: 请求、响应、数据验证、错误信息

## 查看日志

### 方式1：直接查看日志文件
```bash
# Windows PowerShell
Get-Content logs/app.log -Tail 50  # 查看最后50行

# 或使用记事本打开
notepad logs/app.log
```

### 方式2：实时监控日志
```bash
# Windows PowerShell
Get-Content logs/app.log -Wait -Tail 20
```

### 方式3：查看特定错误
```bash
# 查看所有ERROR级别的日志
Select-String "ERROR" logs/app.log

# 查看特定策略的日志
Select-String "MorningStarStrategy" logs/app.log
```

## 诊断步骤

### 问题：连接重置错误 (ERR_CONNECTION_RESET)

**步骤1：检查后端是否启动**
- 查看 `logs/app.log` 是否有"Web服务器启动"日志
- 如果没有，说明后端启动失败

**步骤2：查看启动错误**
- 查看日志中是否有 ERROR 级别的日志
- 查看控制台输出是否有错误信息

**步骤3：检查选股请求**
- 查看日志中是否有"选股请求开始"
- 查看是否有"选股执行失败"的错误

**步骤4：查看详细错误**
- 搜索 "ERROR" 或 "异常" 关键字
- 查看完整的错误堆栈信息

### 问题：选股超时

**步骤1：检查数据加载**
- 查看"加载了 X 只股票代码"
- 查看"成功加载 X 只股票的K线数据"
- 如果数据加载很慢，可能是磁盘I/O问题

**步骤2：检查策略执行**
- 查看"执行策略: XXX"
- 查看"策略 XXX 选中 X 只股票"
- 如果某个策略执行很慢，可能是算法问题

**步骤3：检查交集分析**
- 查看"计算交集分析..."
- 查看"交集分析完成"
- 如果交集分析很慢，可能是数据量太大

## 日志级别说明

| 级别 | 说明 | 示例 |
|------|------|------|
| DEBUG | 调试信息 | 无法读取某只股票的详细原因 |
| INFO | 正常信息 | 选股请求开始、策略执行完成 |
| WARNING | 警告信息 | 无法读取某只股票、分析失败 |
| ERROR | 错误信息 | 选股执行失败、异常堆栈 |

## 常见问题

### Q: 日志文件太大怎么办？
A: 日志系统会自动轮转，单个文件超过10MB时会自动备份，保留最近5个文件。

### Q: 如何只查看ERROR日志？
A: 使用 `Select-String "ERROR" logs/app.log` 命令。

### Q: 如何清空日志？
A: 删除 `logs/app.log` 文件，下次启动时会自动创建新文件。

### Q: 日志中看不到选股请求？
A: 检查后端是否正常启动，查看是否有启动错误。

## 总结

日志系统已完全配置，所有关键操作都有记录。当出现问题时，首先查看 `logs/app.log` 文件，根据日志信息快速定位问题原因。
