# 日志记录改进说明

## 目的

添加详细的日志记录，改进系统的错误诊断能力，便于快速定位和解决问题。

## 改进内容

### 后端日志改进（web_server.py）

#### 1. 选股请求日志
```python
logger.info(f"选股请求 - 策略: {strategies_to_run}, 逻辑: {logic}")
```
- 记录用户选择的策略和逻辑类型
- 便于追踪用户操作

#### 2. 数据加载日志
```python
logger.info(f"加载了 {len(stock_codes)} 只股票代码")
logger.info(f"加载了 {len(stock_names)} 只股票名称")
logger.info(f"成功加载 {len(stock_data)} 只股票的K线数据")
```
- 记录数据加载的进度
- 便于诊断数据加载问题

#### 3. 策略执行日志
```python
logger.info(f"执行策略: {strategy_name}")
logger.info(f"策略 {strategy_name} 选中 {len(signals)} 只股票")
```
- 记录每个策略的执行情况
- 便于诊断策略执行问题

#### 4. 交集分析日志
```python
logger.info(f"多策略结果 - 总策略数: {len(results)}, 有结果: {has_results}")
logger.info("计算交集分析...")
logger.info(f"交集分析完成 - 总股票数: {intersection_analysis.get('total', 0)}")
```
- 记录交集分析的执行情况
- 便于诊断交集分析问题

#### 5. 错误日志
```python
logger.error(f"选股执行失败: {error_msg}")
logger.error(f"错误堆栈: {traceback.format_exc()}")
```
- 记录完整的错误信息和堆栈跟踪
- 便于快速定位错误原因

### 前端日志改进（web/static/js/app.js）

#### 1. 请求日志
```javascript
console.log('选股请求开始', { strategies, logic });
console.log('发送请求体:', JSON.stringify(requestBody));
```
- 记录选股请求的开始和请求体
- 便于追踪前端操作

#### 2. 响应日志
```javascript
console.log('响应状态码:', response.status);
console.log('响应Content-Type:', contentType);
console.log('响应数据:', result);
```
- 记录响应的状态码和内容类型
- 便于诊断网络问题

#### 3. 数据验证日志
```javascript
console.log('选股成功，数据类型:', typeof result.data);
console.log('选股结果键:', Object.keys(result.data || {}));
```
- 记录返回数据的类型和结构
- 便于诊断数据格式问题

#### 4. 错误日志
```javascript
console.error('选股失败:', result.error);
console.error('选股异常:', error);
console.error('错误堆栈:', error.stack);
```
- 记录完整的错误信息和堆栈跟踪
- 便于快速定位错误原因

## 日志级别

### 后端
- **INFO**：正常的操作流程日志
- **WARNING**：警告信息（如无法读取某只股票）
- **ERROR**：错误信息（如选股执行失败）

### 前端
- **log**：正常的操作流程日志
- **error**：错误信息

## 使用场景

### 场景1：诊断选股失败
1. 查看后端日志，找到"选股执行失败"的错误信息
2. 查看错误堆栈，定位具体的错误位置
3. 查看前端日志，确认请求是否正确发送
4. 查看响应数据，确认返回的数据格式是否正确

### 场景2：诊断数据加载问题
1. 查看后端日志中的"加载了X只股票"信息
2. 如果数据加载数量异常，查看警告日志
3. 检查是否有无法读取的股票

### 场景3：诊断策略执行问题
1. 查看后端日志中的"执行策略"信息
2. 查看每个策略选中的股票数量
3. 如果某个策略没有选中任何股票，检查策略参数

## 日志输出位置

### 后端
- 控制台输出（开发环境）
- 日志文件（生产环境，需要配置）

### 前端
- 浏览器开发者工具的控制台（F12 → Console）

## 查看日志的方法

### 后端日志
```bash
# 查看实时日志
tail -f app.log

# 查看特定错误
grep "ERROR" app.log

# 查看特定策略的日志
grep "MorningStarStrategy" app.log
```

### 前端日志
1. 打开浏览器开发者工具（F12）
2. 切换到"Console"标签
3. 执行选股操作
4. 查看输出的日志信息
