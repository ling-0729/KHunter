# 策略执行算法说明

## 概述

系统支持多策略组合执行，通过 OR 和 AND 逻辑来处理多个策略的选股结果。

## 执行流程

### 1. 数据加载阶段
- 加载所有股票代码
- 读取股票名称映射表（stock_names.json）
- 加载每只股票的历史K线数据（CSV格式）
- 过滤：只保留数据充足的股票（至少60条K线）

### 2. 策略执行阶段

#### OR 逻辑（默认）
**目标**：并集 - 任意策略选中即可

**执行流程**：
```
1. 遍历所有策略（或指定的策略列表）
2. 对每个策略：
   - 遍历所有股票
   - 调用 strategy.analyze_stock(code, name, df)
   - 如果返回结果，添加到该策略的信号列表
3. 返回每个策略的独立结果
4. 计算交集分析（可选）
```

**代码位置**：`web_server.py` 的 `run_selection()` 函数，else 分支

**示例**：
```
策略A选中：[股票1, 股票2, 股票3]
策略B选中：[股票2, 股票4, 股票5]
策略C选中：[股票1, 股票5, 股票6]

OR结果：
- 策略A: [股票1, 股票2, 股票3]
- 策略B: [股票2, 股票4, 股票5]
- 策略C: [股票1, 股票5, 股票6]
- 交集分析：
  - 被2个策略选中：[股票2, 股票5]
  - 被3个策略选中：[股票1]
```

#### AND 逻辑（交集）
**目标**：交集 - 所有选中的策略都必须选中该股票

**执行流程**：
```
1. 遍历所有指定的策略
2. 对每个策略执行选股，获得该策略的选中股票集合
3. 计算所有策略选中股票的交集
4. 返回交集结果
```

**代码位置**：`web_server.py` 的 `run_selection()` 函数，if logic == 'and' 分支

**示例**：
```
策略A选中：[股票1, 股票2, 股票3]
策略B选中：[股票2, 股票4, 股票5]
策略C选中：[股票1, 股票5, 股票6]

AND结果（3个策略都选中）：
- 交集：[] （没有股票被所有3个策略都选中）

AND结果（策略A和B）：
- 交集：[股票2]
```

## 核心函数说明

### run_selection()
**位置**：`web_server.py` Line 158

**参数**：
- 请求方法：GET 或 POST
- POST 数据：
  ```json
  {
    "strategies": ["MorningStarStrategy", "BowlReboundStrategy"],
    "logic": "or"  // 或 "and"
  }
  ```

**返回**：
```json
{
  "success": true,
  "data": {
    "MorningStarStrategy": [...],
    "BowlReboundStrategy": [...],
    "_intersection_analysis": {...}  // 仅在OR逻辑且多策略时返回
  },
  "time": "2024-03-19 10:30:45"
}
```

### analyze_intersection()
**位置**：`web_server.py` Line 133

**功能**：分析多策略选股结果的交集

**输入**：
```python
{
  "strategy_name": [
    {"code": "600000", "name": "浦发银行", "signals": {...}},
    ...
  ],
  ...
}
```

**输出**：
```python
{
  "total": 100,  # 总共被选中的股票数
  "by_count": {
    1: [...],  # 被1个策略选中的股票
    2: [...],  # 被2个策略选中的股票
    3: [...]   # 被3个策略选中的股票
  },
  "intersection_stats": {
    "total_strategies": 3,
    "stocks_by_strategy": {
      "MorningStarStrategy": 50,
      "BowlReboundStrategy": 45,
      "B1PatternMatch": 40
    },
    "intersection_rate": 0.25  # 被多个策略选中的比例
  }
}
```

## 策略执行的关键步骤

### 1. 单个策略执行
```python
result = strategy.analyze_stock(code, name, df)
```

**返回格式**：
```python
{
  "code": "600000",
  "name": "浦发银行",
  "signals": {
    "signal_type": "morning_star",
    "confidence": 0.85,
    ...
  }
}
```

### 2. 错误处理
- 无法读取的股票：跳过，继续处理下一只
- 策略分析失败：记录警告，继续处理下一只
- 数据不足的股票：过滤掉（K线数据 < 60）

### 3. 结果聚合
- OR 逻辑：每个策略独立返回结果
- AND 逻辑：返回交集结果（_intersection 字段）

## 性能考虑

### 时间复杂度
- 单策略执行：O(n)，其中 n 为股票数量
- 多策略 OR：O(m × n)，其中 m 为策略数量
- 多策略 AND：O(m × n)，需要额外的交集计算 O(m × k)，其中 k 为平均选中股票数

### 优化建议
1. 缓存股票数据，避免重复读取
2. 并行执行多个策略
3. 对大量股票使用分批处理

## 使用示例

### 执行所有策略（OR逻辑）
```bash
GET /api/selection
```

### 执行指定策略（OR逻辑）
```bash
POST /api/selection
{
  "strategies": ["MorningStarStrategy", "BowlReboundStrategy"],
  "logic": "or"
}
```

### 执行指定策略（AND逻辑）
```bash
POST /api/selection
{
  "strategies": ["MorningStarStrategy", "BowlReboundStrategy"],
  "logic": "and"
}
```

## 注意事项

1. **AND 逻辑的限制**：
   - 需要至少2个策略
   - 如果只有1个策略，自动转为 OR 逻辑
   - 交集可能为空（没有股票被所有策略都选中）

2. **性能影响**：
   - 策略数量越多，执行时间越长
   - 股票数量越多，执行时间越长
   - 建议在后台异步执行

3. **数据一致性**：
   - 每次执行都会重新加载股票数据
   - 参数修改后立即生效
   - 不存在缓存问题
