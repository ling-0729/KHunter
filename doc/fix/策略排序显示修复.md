# 策略排序显示修复

## 问题描述

策略配置页面和执行选股页面没有按照 `config/strategy_order.yaml` 中定义的顺序展示策略。

## 根本原因

后端的 `/api/strategies` 接口中，get_strategies函数使用了硬编码的策略顺序列表，而不是从 `config/strategy_order.yaml` 中读取的排序信息。

## 解决方案

修改 `web_server.py` 中的 `get_strategies()` 函数，使其使用 `registry.list_strategies()` 方法获取按排序顺序的策略列表。

### 修改前

```python
# 定义策略显示顺序（按照YAML中的定义顺序）
strategy_order = [
    'B1PatternMatch',
    'BowlReboundStrategy',
    'MorningStarStrategy',
    'BottomTrendInflectionStrategy',
    'TrendAccelerationInflectionStrategy'
]

# 按照定义的顺序遍历策略
for name in strategy_order:
    # ... 处理策略 ...

# 添加任何未在定义顺序中的策略
for name in registry.strategies.keys():
    if name not in strategy_order:
        # ... 处理策略 ...
```

### 修改后

```python
# 使用registry中的排序信息（从strategy_order.yaml加载）
# 按照排序顺序获取策略名称
sorted_strategy_names = registry.list_strategies()

# 按照排序顺序遍历策略
for name in sorted_strategy_names:
    if name not in registry.strategies:
        continue
    # ... 处理策略 ...
```

## 工作流程

1. 后端启动时，`StrategyRegistry` 初始化时会从 `config/strategy_order.yaml` 加载排序配置
2. `registry._load_strategy_order()` 方法将排序配置转换为 `{策略名: 排序号}` 的映射
3. 当前端请求 `/api/strategies` 时，调用 `registry.list_strategies()` 获取按排序顺序的策略列表
4. 后端按照这个顺序返回策略数据
5. 前端接收到的策略列表已经是正确的顺序，直接渲染即可

## 影响范围

- **策略配置页面** (`/strategies`): 策略卡片现在按照 `strategy_order.yaml` 中定义的顺序展示
- **执行选股页面** (策略选择对话框): 策略列表现在按照 `strategy_order.yaml` 中定义的顺序展示
