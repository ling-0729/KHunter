# 选股结果中文策略名称修复

## 问题描述

选股结果保存时，策略名称保存的是英文（如 `MorningStarStrategy`），而不是配置文件中定义的中文名称（如 `启明星策略`）。

## 根本原因

`utils/selection_record_manager.py` 中的 `generate_strategy_name()` 方法使用了硬编码的旧映射：
```python
strategy_display_names = {
    'morning_star': '晨星',
    'bowl_rebound': '碗形反弹',
    'pattern_match': '形态匹配'
}
```

但实际传入的策略名称是新的英文名称（`MorningStarStrategy`、`BowlReboundStrategy`、`B1PatternMatch`），导致映射失败，最终保存的还是英文名称。

## 解决方案

修改 `generate_strategy_name()` 方法，从 `config/strategy_params.yaml` 配置文件中动态读取每个策略的 `display_name` 字段，而不是使用硬编码的映射。

### 修改内容

**文件**: `utils/selection_record_manager.py`

**方法**: `generate_strategy_name()`

**修改前**:
- 使用硬编码的旧映射
- 映射不完整且容易过时
- 新增策略需要手动修改代码

**修改后**:
- 从配置文件动态读取中文名称
- 自动适配所有策略
- 配置文件更新时自动生效

### 映射关系

| 英文策略名 | 中文名称 |
|-----------|--------|
| MorningStarStrategy | 启明星策略 |
| BowlReboundStrategy | 碗口反弹策略 |
| B1PatternMatch | B1形态匹配 |
