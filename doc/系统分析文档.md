# Kunter 系统分析文档

## 一、项目概述

**项目名称**：Kunter  
**项目类型**：A股量化选股系统  
**核心功能**：基于多种技术分析策略的自动化选股系统  
**技术栈**：Python 3.8+ + Flask + SQLite + akshare  
**目标用户**：量化投资者、股票分析师、投资爱好者  

### 项目定位
Kunter是一个完整的A股量化选股解决方案，集数据获取、策略分析、选股执行于一体。系统支持12种不同的选股策略，可灵活组合使用。

## 三、核心功能模块详解

### 3.1 策略模块（12种选股策略）

#### 策略列表

| # | 策略名称 | 文件 | 核心逻辑 | 参数数量 |
|----|---------|------|--------|--------|
| 1 | 碗口反弹策略 | bowl_rebound.py | 知行趋势线与多空线的碗口反弹形态 | 10 |
| 2 | B1完美图形匹配 | pattern_*.py | 基于10个历史案例的三维相似度匹配 | 5 |
| 3 | 启明星策略 | morning_star.py | 三根K线底部反转形态 | 4 |
| 4 | 底部趋势拐点 | bottom_trend_inflection.py | 深度下跌后的反转拐点 | 6 |
| 5 | 趋势加速拐点 | trend_acceleration_inflection.py | 上升趋势中的加速拐点 | 7 |
| 6 | 多金叉共振 | multi_golden_cross.py | 均线/KDJ/MACD金叉共振 | 8 |
| 7 | 多死叉共振 | multi_death_cross.py | 均线/KDJ/MACD死叉共振 | 8 |
| 8 | 多方炮策略 | multi_party_cannon.py | 两阳夹一阴K线组合 | 13 |
| 9 | 缩量回调策略 | volume_shrinkage_pullback.py | 上升趋势中的缩量回调 | 11 |
| 10 | 阻力位突破策略 | resistance_breakout.py | 股价突破关键阻力位 | 8 |
| 11 | W底策略 | w_bottom_strategy.py | W底双底反转形态 | 11 |
| 12 | M头策略 | m_top_strategy.py | M头双顶反转形态 | 11 |

#### 策略架构
- **BaseStrategy** - 策略基类，定义策略接口
  - `calculate_indicators()` - 计算技术指标
  - `select_stocks()` - 执行选股逻辑
  - `analyze_stock()` - 单只股票分析
- **StrategyRegistry** - 策略注册器
  - 自动发现和注册策略
  - 支持动态加载策略参数
  - 支持策略的启用/禁用

### 3.2 数据获取模块

**主要功能**：
- 获取全部A股代码（5000+只）
- 获取历史K线数据（支持6年回溯）
- 增量更新数据（智能检测）
- 获取实时价格和市值
- 批量数据获取（支持并发）

**数据源优先级**：
1. akshare API（主要数据源）
2. HTTP接口（备选）
3. 本地缓存（降级）
4. 模拟数据（最后降级）

**智能更新策略**：
- 3点前不更新（使用本地数据）
- 3点后检查每只股票是否有当天数据
- 100%有当天数据则跳过更新
- 否则执行增量更新

### 3.3 技术指标模块

**支持的技术指标**：
- **KDJ** - 随机指标（K、D、J值）
- **EMA** - 指数移动平均
- **MA** - 简单移动平均（MA5, MA10, MA14, MA20, MA28, MA57, MA114）
- **MACD** - 指数平滑异同移动平均线
- **知行短期趋势线** - EMA(EMA(CLOSE, 10), 10)
- **知行多空线** - (MA14 + MA28 + MA57 + MA114) / 4

### 3.4 交易模块

**功能**：模拟交易系统，支持虚拟账户管理

**主要类**：
- **TradingAccountDAO** - 账户管理
- **TradingPositionDAO** - 持仓管理
- **TradingTransactionDAO** - 交易记录
- **TradingService** - 业务逻辑
- **TradingCalculator** - 计算器（成本价、手续费、收益）
- **TradingValidator** - 验证器（T+1规则、资金验证）

**支持的操作**：
- 买入/卖出
- 账户总览
- 持仓查询
- 交易历史
- 收益计算

### 3.5 Web服务模块

**REST API接口**：
```

GET  /api/trading/positions         - 获取持仓列表
GET  /api/trading/transactions      - 获取交易历史
```

**前端功能**：
- 系统概览（股票数量、最新数据日期、系统状态）
- 股票列表（搜索、分页、基本信息）

## 四、数据库设计

### 4.1 数据库类型
- **SQLite** - 用于选股记录和交易数据
  - stock_selection.db - 选股记录
  - trading.db - 交易数据
- **CSV** - 用于股票K线数据（按代码前缀分类）

### 4.2 主要表结构

**stock_basic** - 股票基础信息
```sql
CREATE TABLE stock_basic (
    id INT PRIMARY KEY,
    code VARCHAR(10) UNIQUE,      -- 股票代码
    name VARCHAR(50),              -- 股票名称
    industry VARCHAR(50),          -- 所属行业
    area VARCHAR(50),              -- 所属地区
    market VARCHAR(20),            -- 市场类型
    list_date DATE,                -- 上市日期
    update_time DATETIME           -- 更新时间
);
```

**trading_account** - 交易账户
```sql
CREATE TABLE trading_account (
    account_id VARCHAR(50) PRIMARY KEY,
    account_name VARCHAR(100),
    initial_cash DECIMAL(15,2),    -- 初始资金
    current_cash DECIMAL(15,2),    -- 当前现金
    total_assets DECIMAL(15,2),    -- 总资产
    total_profit_loss DECIMAL(15,2), -- 总收益
    profit_loss_rate DECIMAL(8,2), -- 收益率(%)
    created_date DATE,
    updated_date DATE,
    status VARCHAR(20)
);
```

**trading_position** - 持仓信息
```sql
CREATE TABLE trading_position (
    position_id VARCHAR(50) PRIMARY KEY,
    account_id VARCHAR(50),
    stock_code VARCHAR(10),
    stock_name VARCHAR(50),
    quantity INT,                  -- 持仓数量
    cost_price DECIMAL(10,4),      -- 成本价
    current_price DECIMAL(10,4),   -- 当前价
    market_value DECIMAL(15,2),    -- 市值
    profit_loss DECIMAL(15,2),     -- 收益
    profit_loss_rate DECIMAL(8,2), -- 收益率(%)
    last_buy_date DATE,
    created_date DATE,
    updated_date DATE
);
```

**trading_transaction** - 交易记录
```sql
CREATE TABLE trading_transaction (
    transaction_id VARCHAR(50) PRIMARY KEY,
    account_id VARCHAR(50),
    stock_code VARCHAR(10),
    stock_name VARCHAR(50),
    transaction_type VARCHAR(10),  -- BUY/SELL
    quantity INT,
    price DECIMAL(10,4),
    amount DECIMAL(15,2),
    commission DECIMAL(10,2),      -- 手续费
    stamp_tax DECIMAL(10,2),       -- 印花税
    total_cost DECIMAL(15,2),
    profit_loss DECIMAL(15,2),
    transaction_date DATE,
    created_date DATETIME
);
```

### 4.3 CSV数据格式
K线数据存储在CSV文件中，按代码前缀分类：
- `data/00/000001.csv` - 平安银行
- `data/30/300001.csv` - 浪潮信息
- `data/60/600519.csv` - 贵州茅台
- `data/68/688001.csv` - 华兴源创

**CSV列结构**：
```
date,open,high,low,close,volume,amount,turnover,market_cap
2024-03-01,10.50,10.65,10.45,10.60,50000000,530000000,2.5,500000000000
```

## 五、系统工作流程

### 5.1 初始化流程
```
main.py init
    ↓
获取全部A股代码（5000+只）
    ↓
批量获取市值数据
    ↓
逐只股票获取6年历史数据
    ↓
保存到CSV文件（按代码前缀分类）
    ↓
记录失败股票列表
```

### 5.2 选股流程
```
main.py run
    ↓
智能更新数据（3点后检查）
    ↓
加载所有策略
    ↓
逐只股票执行策略
    ↓
计算技术指标
    ↓
判断选股条件
    ↓
收集选股结果
```

### 5.3 B1完美图形匹配流程
```
main.py run --b1-match
    ↓
执行原有选股逻辑
    ↓
初始化B1完美图形库（10个案例）
    ↓
对每只候选股进行匹配
    ↓
计算三维相似度
    ↓
按相似度排序
```

## 七、系统特色功能

### 7.1 B1完美图形匹配
- 基于10个历史成功案例
- 三维相似度匹配
  - 双线结构相似度：30%
  - KDJ状态相似度：20%
  - 量能特征相似度：25%
  - 价格形态相似度：25%
- 使用DTW算法计算价格曲线相似度
- 支持自定义回看天数和相似度阈值

### 7.2 智能数据更新
- 3点前不更新（使用本地数据）
- 3点后检查每只股票是否有当天数据
- 100%有当天数据则跳过更新
- 否则执行增量更新

### 7.3 多策略交集分析
- 支持OR逻辑（并集）
- 支持AND逻辑（交集）
- 自动计算交集率和分布

### 7.4 参数锁定和追踪
- 参数修改自动检测
- 参数变化自动恢复
- 参数变化日志记录
