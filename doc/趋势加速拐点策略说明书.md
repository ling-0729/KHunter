# 趋势加速拐点策略说明书

## 策略概述

趋势加速拐点策略是一种识别股票在上升趋势中出现加速信号的量化选股策略。该策略通过识别四个关键条件的组合，找出处于上升趋势中且出现加速信号的股票。

### 策略特点

- **顺势而为**：在上升趋势中选股，避免逆势操作
- **信号明确**：放量长阳线是加速信号，信号清晰
- **多重确认**：四个条件组合确认，降低误信号
- **风险可控**：回调支撑明确，风险可控

## 选股条件

策略通过以下四个条件的组合来识别趋势加速拐点：

### 条件1：上升趋势（近20交易日）

使用线性回归法判断股票是否处于上升趋势，通过统计学方法量化趋势强度和显著性。

**判断方法**：
- 对近20个交易日的收盘价进行线性回归分析
- 计算回归直线的斜率、p值（显著性）和R²（拟合度）
- 上升趋势确认需同时满足三个条件：
  1. **斜率 > 0**：价格总体呈上升趋势
  2. **p值 < 0.01**：趋势具有统计显著性（99%置信度），排除随机波动
  3. **R² > 0.5**：拟合度良好，趋势明显且稳定

**数学原理**：
- 线性回归模型：Y = slope × X + intercept
- X轴：交易日序号（0, 1, 2, ..., n-1）
- Y轴：收盘价
- R²（决定系数）：表示收盘价变化中有多少比例可以由时间解释，范围[0,1]

**参数**：
- `uptrend_lookback_days`: 上升趋势回溯天数（默认20）
- `uptrend_slope_threshold`: 斜率阈值（默认0，表示斜率必须大于0）
- `uptrend_pvalue_threshold`: p值阈值（默认0.01，要求99%置信度）
- `uptrend_rsquared_threshold`: R²阈值（默认0.5，要求较好的拟合度）

### 条件2：放量长阳线（近10交易日）

识别近10个交易日内出现的放量长阳线，作为加速信号。

**判断方法**：
- 涨幅 > 8%（长阳线）
- 成交量 >= 2倍5日均量（放量）

**参数**：
- `price_increase_threshold`: 涨幅阈值（默认0.08 = 8%）
- `volume_ratio_threshold`: 成交量倍数阈值（默认2.0）
- `volume_ma_period`: 成交量均值周期（默认5）
- `surge_lookback_days`: 放量检查周期（默认10）

### 条件3：距离条件（最长回溯40交易日）

长阳线起涨点距离最近最低点的涨幅不能超过15%。

**判断方法**：
- 在最长回溯40交易日内找到最低点
- 计算长阳线起涨点到最低点的涨幅
- 涨幅 < 15%

**参数**：
- `distance_threshold`: 距离阈值（默认0.15 = 15%）
- `lowest_point_lookback_days`: 最低点回溯天数（默认40）

### 条件4：回调支撑

长阳线后的回调没有跌破长阳线的开盘价。

**判断方法**：
- 检查长阳线后的最低价
- 最低价 >= 长阳线开盘价

## 参数配置

### 参数分组

#### 趋势条件
- `uptrend_lookback_days`: 上升趋势回溯天数
  - 默认值：20
  - 范围：10-60
  - 说明：用于线性回归分析判断趋势方向

- `uptrend_slope_threshold`: 斜率阈值
  - 默认值：0
  - 范围：-0.1 ~ 0.1
  - 说明：线性回归斜率必须大于此值，0表示价格总体上升

- `uptrend_pvalue_threshold`: p值阈值
  - 默认值：0.01
  - 范围：0.001 ~ 0.1
  - 说明：统计显著性阈值，越小表示对趋势显著性要求越高（0.01表示99%置信度）

- `uptrend_rsquared_threshold`: R²阈值
  - 默认值：0.5
  - 范围：0.3 ~ 0.9
  - 说明：拟合度阈值，越大表示要求趋势越稳定（0.5表示50%的价格变化可由时间解释）

#### 放量条件
- `price_increase_threshold`: 涨幅阈值(%)
  - 默认值：0.08
  - 范围：0.05-0.3
  - 说明：长阳线的涨幅需要超过此值

- `volume_ratio_threshold`: 成交量倍数
  - 默认值：2.0
  - 范围：1.5-5.0
  - 说明：当日成交量需要是成交量均值的此倍数以上

- `volume_ma_period`: 成交量周期
  - 默认值：5
  - 范围：3-20
  - 说明：成交量均值计算周期

- `surge_lookback_days`: 放量检查天数
  - 默认值：10
  - 范围：5-30
  - 说明：在此天数内寻找放量长阳线

#### 距离条件
- `distance_threshold`: 距离阈值(%)
  - 默认值：0.15
  - 范围：0.05-0.3
  - 说明：长阳线起涨点距离最近最低点的涨幅不能超过此值

- `lowest_point_lookback_days`: 最低点回溯天数
  - 默认值：40
  - 范围：20-120
  - 说明：在此天数内寻找最近的最低点

## 使用指南

### 基本使用

1. **启动系统**
   ```bash
   python web_server.py
   ```

2. **访问前端**
   - 打开浏览器，访问 http://localhost:5000

3. **执行选股**
   - 点击"执行选股"按钮
   - 在策略选择对话框中勾选"趋势加速拐点"
   - 选择选股逻辑（OR 或 AND）
   - 点击"确认"执行选股

4. **查看结果**
   - 在"选股结果"页面查看选出的股票
   - 点击股票可查看详细信息

### 参数调整

#### 激进策略（选股数量多）
- 增加 `uptrend_lookback_days`（如30）
- 提高 `uptrend_pvalue_threshold`（如0.05，降低显著性要求）
- 降低 `uptrend_rsquared_threshold`（如0.3，降低拟合度要求）
- 降低 `price_increase_threshold`（如0.05）
- 降低 `volume_ratio_threshold`（如1.5）
- 增加 `distance_threshold`（如0.25）

#### 保守策略（选股数量少）
- 降低 `uptrend_lookback_days`（如15）
- 降低 `uptrend_pvalue_threshold`（如0.001，提高显著性要求）
- 提高 `uptrend_rsquared_threshold`（如0.7，提高拟合度要求）
- 增加 `price_increase_threshold`（如0.12）
- 增加 `volume_ratio_threshold`（如3.0）
- 降低 `distance_threshold`（如0.10）

## 参数调整指南

### 调整思路

1. **选股数量过多**
   - 提高 `price_increase_threshold`（涨幅要求更高）
   - 提高 `volume_ratio_threshold`（成交量要求更高）
   - 降低 `distance_threshold`（距离要求更严格）
   - 降低 `uptrend_pvalue_threshold`（如0.001，提高趋势显著性要求）
   - 提高 `uptrend_rsquared_threshold`（如0.7，提高趋势稳定性要求）

2. **选股数量过少**
   - 降低 `price_increase_threshold`（涨幅要求降低）
   - 降低 `volume_ratio_threshold`（成交量要求降低）
   - 提高 `distance_threshold`（距离要求放松）
   - 提高 `uptrend_pvalue_threshold`（如0.05，降低趋势显著性要求）
   - 降低 `uptrend_rsquared_threshold`（如0.3，降低趋势稳定性要求）

3. **选股质量不佳（趋势不明确）**
   - 增加 `uptrend_lookback_days`（确保真正的上升趋势）
   - 降低 `uptrend_pvalue_threshold`（提高统计显著性要求）
   - 提高 `uptrend_rsquared_threshold`（提高趋势稳定性要求）
   - 增加 `volume_ma_period`（使用更长周期的成交量均值）
   - 增加 `lowest_point_lookback_days`（寻找更远的最低点）

### 参数优化建议

- 建议先使用默认参数运行，观察选股结果
- 根据实际选股数量和质量，逐步调整参数
- 每次调整一个参数，观察效果
- 避免过度优化，防止过拟合

## 常见问题

### Q1: 为什么没有选出任何股票？

**可能原因**：
- 参数设置过于严格
- 市场中没有符合条件的股票
- 数据不完整

**解决方案**：
- 降低 `price_increase_threshold` 和 `volume_ratio_threshold`
- 增加 `distance_threshold`
- 提高 `uptrend_pvalue_threshold`（如0.05，降低趋势显著性要求）
- 降低 `uptrend_rsquared_threshold`（如0.3，降低拟合度要求）
- 检查数据是否完整

### Q2: 选出的股票太多，如何筛选？

**解决方案**：
- 提高 `price_increase_threshold`
- 提高 `volume_ratio_threshold`
- 降低 `distance_threshold`
- 降低 `uptrend_pvalue_threshold`（如0.001，提高趋势显著性要求）
- 提高 `uptrend_rsquared_threshold`（如0.7，提高趋势稳定性要求）
- 结合其他指标进行二次筛选

### Q3: 策略的准确率如何？

**说明**：
- 策略基于技术面分析，准确率取决于市场环境
- 建议结合基本面分析进行综合判断
- 建议设置止损点，控制风险

## 技术细节

### 数据要求

- 最少50个交易日的日线数据
- 包含：开盘价、最高价、最低价、收盘价、成交量

### 计算流程

1. **数据准备**
   - 加载股票日线数据（开盘价、最高价、最低价、收盘价、成交量）
   - 验证数据完整性和有效性

2. **指标计算**
   - 计算KDJ指标（K、D、J值）
   - 计算趋势线（短期趋势、牛熊线）
   - 计算成交量均值（5日均量）

3. **条件检查**
   - **条件1：上升趋势判断**
     - 对近20日收盘价进行线性回归分析
     - 计算斜率、p值、R²
     - 确认：斜率>0 AND p值<0.01 AND R²>0.5
   - **条件2：放量长阳线识别**
     - 在近10个交易日内寻找涨幅>8%的长阳线
     - 确认成交量≥2倍5日均量
   - **条件3：距离条件验证**
     - 计算长阳线起涨点距离最近最低点的涨幅
     - 确认涨幅<15%
   - **条件4：回调支撑检查**
     - 检查长阳线后的最低价是否跌破长阳线开盘价
     - 确认回调有支撑

4. **结果过滤**
   - 过滤 ST、*ST、退市股票
   - 生成选股信号和理由

5. **返回结果**
   - 返回符合条件的股票列表

- 内存占用 < 500MB
- 支持实时更新
