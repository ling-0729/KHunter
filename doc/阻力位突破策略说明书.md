# 阻力位突破策略说明书

## 策略概述

阻力位突破策略是一种识别股票放量长阳突破关键阻力位信号的量化选股策略。该策略通过阻力位识别、放量长阳突破、回踩支撑、趋势配合四个关键条件的组合，找出突破关键价位且站稳的股票。

### 策略特点

- **突破确认**：识别放量长阳突破前期高点阻力位的信号
- **多维验证**：结合价格突破、成交量放大、回踩支撑、趋势方向四个维度
- **严格筛选**：四个条件必须同时满足，确保选股质量
- **风险控制**：通过回踩不破开盘价确认突破有效性

---

## 选股规则详解

### 规则1：阻力位识别

**规则描述**: 计算前60个交易日的最高价作为阻力位

**详细说明**:
- **时间范围**: 回溯 60 个交易日（约3个月）
- **阻力位定义**: 突破日之前60个交易日内出现的最高价
- **计算方式**: 取突破日之前60天的 `high` 最大值

**实际含义**:
- 阻力位是前期多空博弈的关键价位
- 股价在此位置曾遇到卖压而回落
- 突破阻力位意味着多头力量占优

**参数**:
- `lookback_days`: 60 (交易日)

---

### 规则2：放量长阳突破（突破日搜索）

**规则描述**: 在最近10个交易日内搜索同时满足涨幅、阻力位、放量三个条件的突破日

**详细说明**:

#### 2.1 涨幅条件
- **要求**: 突破日涨幅 >= 8%
- **计算**: 涨幅 = (收盘价 - 开盘价) / 开盘价
- **说明**: 使用当日开盘价和收盘价计算，确保是当日的放量长阳

#### 2.2 阻力位突破条件
- **要求**: 突破日收盘价 >= 前60日最高价 × 0.98
- **计算**: 阻力位 = 突破日之前60天的最高价
- **判断**: 收盘价 >= 阻力位 × (1 + breakout_ratio)，breakout_ratio = -0.02
- **说明**: 达到阻力位98%即视为突破，允许小幅误差

#### 2.3 成交量条件
- **要求**: 突破日成交量 >= 前10日均量 × 2.0
- **计算**: 成交量比 = 突破日成交量 / 前10日平均成交量
- **说明**: 前10日平均成交量不包括突破日当天

#### 2.4 搜索逻辑
- **搜索范围**: 最近10个交易日
- **搜索方向**: 从最近的交易日往前搜索，优先找最近的突破
- **判断**: 三个条件同时满足才认定为突破日

**实际含义**:
- 放量长阳说明多头力量强劲
- 突破阻力位说明前期卖压已被消化
- 成交量放大说明市场认可度高
- 三者结合确认突破的真实性和有效性

**参数**:
- `min_change_pct`: 0.08 (8%)
- `breakout_ratio`: -0.02 (达到阻力位98%即可)
- `volume_ratio`: 2.0 (成交量倍数)
- `volume_ma_period`: 10 (成交量均值周期)
- `max_search_days`: 10 (最大搜索天数)

---

### 规则3：回踩支撑

**规则描述**: 从突破日次日到今天，所有交易日的最低价不跌破突破日开盘价

**详细说明**:
- **支撑位定义**: 突破日的开盘价
- **检查范围**: 突破日次日到最新交易日
- **判断逻辑**:
  - 获取突破日的开盘价作为支撑位
  - 检查突破后每天的最低价
  - 如果任何一天最低价跌破突破日开盘价，则不满足条件
- **特殊情况**: 如果突破日就是今天（最新交易日），无需检查回踩，直接通过

**实际含义**:
- 突破日开盘价是多空博弈的起点
- 回踩不破开盘价说明突破力度强，多头控盘
- 跌破开盘价说明突破可能是假突破
- 确保选出的是真正站稳阻力位的股票

---

### 规则4：趋势配合

**规则描述**: 确认短期趋势向上，满足以下任一条件即可

**详细说明**:
- **条件A**: 短期趋势线在多空线上方（趋势线 > 多空线）
- **条件B**: 短期趋势线方向向上（当前值 > 前一天值）
- **趋势线计算**: 使用知行趋势线组件，参数 m1=14, m2=28, m3=57, m4=114

**实际含义**:
- 趋势向上说明大方向配合突破
- 避免在下跌趋势中选择假突破
- 趋势线拐头向上也是有效信号

## 过滤条件

### 过滤1：退市股票
- 过滤包含"退"、"未知"、"退市"、"已退"等关键词的股票

### 过滤2：ST 股票
- 过滤以"ST"或"*ST"开头的股票

### 过滤3：市值过滤
- 最小市值：20亿元（过滤小盘股）
- 最大市值：1000亿元（过滤超大盘股）

### 过滤4：数据有效性
- 股票必须有至少70个交易日的历史数据（lookback_days + max_search_days）
- 最新一天必须有有效的成交量和收盘价

## 参数配置

### 参数分组

#### 阻力位参数
- `lookback_days`: 回溯天数
  - 默认值：60
  - 范围：30-120
  - 说明：用于计算阻力位的时间窗口（60个交易日约3个月）

- `breakout_ratio`: 突破阈值
  - 默认值：-0.02
  - 范围：-0.05 ~ 0.05
  - 说明：-0.02表示达到阻力位98%即视为突破，正值表示必须超过阻力位

#### 突破日条件
- `min_change_pct`: 突破日最小涨幅
  - 默认值：0.08
  - 范围：0.05-0.15
  - 说明：突破日涨幅需要超过此值（0.08表示8%）

- `volume_ratio`: 成交量倍数
  - 默认值：2.0
  - 范围：1.5-5.0
  - 说明：突破日成交量需要是均量的此倍数以上

- `volume_ma_period`: 成交量均值周期
  - 默认值：10
  - 范围：5-20
  - 说明：计算成交量均值的天数

#### 搜索参数
- `max_search_days`: 最大搜索天数
  - 默认值：10
  - 范围：5-20
  - 说明：在最近N天内搜索突破日

#### 市值过滤
- `min_market_cap`: 最小市值（亿元）
  - 默认值：20
  - 范围：10-100
  - 说明：过滤市值过小的股票

- `max_market_cap`: 最大市值（亿元）
  - 默认值：1000
  - 范围：500-5000
  - 说明：过滤市值过大的股票

   ```bash
   python web_server.py
   ```

   - 点击"确认"执行选股

4. **查看结果**
   - 在"选股结果"页面查看选出的股票
   - 点击股票可查看详细信息

### 参数调整

#### 激进策略（选股数量多）
- 降低 `breakout_ratio`（如-0.05，允许更大偏差）
- 降低 `min_change_pct`（如0.05，降低涨幅要求）
- 降低 `volume_ratio`（如1.5，降低成交量要求）
- 增加 `max_search_days`（如15，扩大搜索范围）
- 增加 `lookback_days`（如90，延长阻力位计算窗口）

#### 保守策略（选股数量少）
- 提高 `breakout_ratio`（如0.02，要求超过阻力位2%）
- 提高 `min_change_pct`（如0.10，提高涨幅要求）
- 提高 `volume_ratio`（如3.0，提高成交量要求）
- 降低 `max_search_days`（如5，缩小搜索范围）
- 降低 `lookback_days`（如30，缩短阻力位计算窗口）

## 参数调整指南

### 调整思路

1. **选股数量过多**
   - 提高 `breakout_ratio`（要求更高的突破幅度）
   - 提高 `min_change_pct`（涨幅要求更高）
   - 提高 `volume_ratio`（成交量要求更高）
   - 降低 `max_search_days`（缩小搜索范围）

2. **选股数量过少**
   - 降低 `breakout_ratio`（允许更大偏差）
   - 降低 `min_change_pct`（涨幅要求降低）
   - 降低 `volume_ratio`（成交量要求降低）
   - 增加 `max_search_days`（扩大搜索范围）
   - 增加 `lookback_days`（延长阻力位计算窗口）

3. **选股质量不佳（假突破多）**
   - 提高 `breakout_ratio`（如0.02，要求超过阻力位）
   - 提高 `volume_ratio`（确保成交量充分放大）
   - 降低 `max_search_days`（只看最近几天的突破）
   - 提高 `min_change_pct`（确保突破力度够大）

### 参数优化建议

- 建议先使用默认参数运行，观察选股结果
- 根据实际选股数量和质量，逐步调整参数
- 每次调整一个参数，观察效果
- 避免过度优化，防止过拟合
- `breakout_ratio` 是最敏感的参数，小幅调整即可产生明显效果

## 常见问题

### Q1: 为什么没有选出任何股票？

**可能原因**：
- 当前市场环境下满足全部4个条件的股票较少
- 参数设置过于严格
- 数据不完整

**解决方案**：
- 降低 `min_change_pct`（如从0.08降至0.05）
- 降低 `volume_ratio`（如从2.0降至1.5）
- 降低 `breakout_ratio`（如从-0.02降至-0.05）
- 增加 `max_search_days`（如从10增至15）
- 检查数据是否完整

### Q2: 选出的股票太多，如何筛选？

**解决方案**：
- 提高 `breakout_ratio`（如从-0.02提高至0.02）
- 提高 `min_change_pct`（如从0.08提高至0.10）
- 提高 `volume_ratio`（如从2.0提高至3.0）
- 降低 `max_search_days`（如从10降至5）
- 结合其他策略进行交集筛选

### Q3: 如何理解 breakout_ratio 参数？

**说明**：
- `breakout_ratio = -0.02` 表示收盘价达到阻力位的98%即视为突破
- `breakout_ratio = 0` 表示收盘价必须等于或超过阻力位
- `breakout_ratio = 0.02` 表示收盘价必须超过阻力位2%
- 负值更宽松（容易选出股票），正值更严格（选出的突破更确定）

### Q4: 为什么用开盘价作为回踩支撑位？

**说明**：
- 突破日的开盘价是当天多空博弈的起点
- 放量长阳从开盘价拉升到收盘价，开盘价是多头的成本线
- 回踩不破开盘价说明多头力量强劲，突破有效
- 跌破开盘价说明突破力度不够，可能是假突破

### Q5: 四个条件的优先级如何？

**说明**：
- **规则1（阻力位识别）**：基础条件，确定关键价位
- **规则2（放量长阳突破）**：核心条件，确认突破信号
- **规则3（回踩支撑）**：验证条件，确认突破有效性
- **规则4（趋势配合）**：辅助条件，确认大方向配合
- **建议**：规则2是最关键的，规则3和规则4可以适当放宽

## 技术细节

### 数据要求
- 最少70个交易日的日线数据（lookback_days + max_search_days）
- 包含：日期、开盘价、最高价、最低价、收盘价、成交量

### 计算流程

1. **数据准备**
   - 加载股票日线数据
   - 验证数据完整性和有效性
   - 确保数据按时间正序排列（最新在后）

2. **指标计算**
   - 计算阻力位（前60日最高价滚动窗口）
   - 计算成交量均线（10日均量）
   - 计算成交量比（当日成交量 / 10日均量）
   - 计算知行趋势线（短期趋势线、多空线）
   - 估算市值

3. **条件检查**
   - **突破日搜索**：在最近10天内从后往前搜索，找到第一个同时满足涨幅>=8%、收盘价>=阻力位×0.98、成交量>=2倍均量的交易日
   - **回踩检查**：突破日次日到今天，所有天最低价不跌破突破日开盘价
   - **趋势检查**：短期趋势线在多空线上方，或趋势线方向向上

4. **结果过滤**
   - 过滤 ST、*ST、退市股票
   - 过滤市值不在20-1000亿范围的股票
   - 生成选股信号和理由

5. **返回结果**
   - 返回符合条件的股票列表

### 性能指标

- 处理500只股票耗时 < 10秒
- 内存占用 < 500MB
- 支持实时更新

## 选股流程

```
输入：股票日线数据（最新 70+ 个交易日）
  ↓
第1步：数据有效性检查
  - 检查数据长度 ≥ 70 天
  - 检查必要字段完整
  - 检查最新一天有有效成交量和收盘价
  ↓
第2步：过滤检查
  - 过滤退市股票
  - 过滤 ST 股票
  - 过滤市值不在 20-1000 亿范围的股票
  ↓
第3步：突破日搜索（规则1+规则2）
  - 在最近 10 天内从后往前搜索
  - 检查涨幅 >= 8%（放量长阳）
  - 检查收盘价 >= 前60日最高价 × 0.98（突破阻力位）
  - 检查成交量 >= 前10日均量 × 2.0（放量确认）
  - 三个条件同时满足 → 找到突破日
  ↓
第4步：回踩检查（规则3）
  - 获取突破日开盘价作为支撑位
  - 检查突破后每天最低价是否跌破支撑位
  - 未跌破 → 通过
  ↓
第5步：趋势检查（规则4）
  - 检查短期趋势线是否在多空线上方
  - 或检查短期趋势线是否拐头向上
  - 满足任一 → 通过
  ↓
第6步：生成信号
  - 四个条件都满足 → 生成选股信号
  - 任何条件不满足 → 不选股
  ↓
输出：选股信号（包含阻力位、突破幅度、量比、趋势等信息）
```

## 输出信息

当股票满足所有条件时，输出以下信息：

| 字段 | 说明 |
|------|------|
| `date` | 选股日期 |
| `close` | 最新收盘价 |
| `resistance_level` | 阻力位（前60日最高价） |
| `breakout_ratio` | 突破幅度（突破日收盘价相对阻力位的比例） |
| `volume` | 最新成交量 |
| `volume_ma` | 10日成交量均值 |
| `volume_ratio` | 成交量比（最新成交量 / 10日均量） |
| `short_term_trend` | 短期趋势线 |
| `bull_bear_line` | 多空线 |
| `market_cap` | 市值（亿元） |
| `breakout_date` | 突破日日期 |
| `days_since_breakout` | 突破后经过的天数 |
| `reasons` | 选股原因列表 |
| `pattern_details` | 详细模式信息 |

## 策略特点

### 优点
1. **逻辑清晰**: 突破阻力位是经典的技术分析信号，逻辑直观易懂
2. **多维验证**: 价格突破、成交量放大、回踩支撑、趋势配合四重确认
3. **假突破过滤**: 回踩不破开盘价有效过滤假突破信号
4. **时效性强**: 只搜索最近10天的突破，确保信号新鲜

### 缺点
1. **选股严格**: 四个条件同时满足的股票较少，选股比例约0.4%
2. **市场环境依赖**: 震荡市或下跌市中突破信号较少
3. **追高风险**: 突破后买入可能面临短期回调
