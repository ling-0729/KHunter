# Web页面数据更新功能测试

## 测试目标
验证Web页面上的手动数据更新功能是否正常工作

## 测试环境
- 系统: Windows
- Python: 3.12.10
- Flask: 3.0.3
- 浏览器: Chrome/Edge

## 测试用例

### TC-001: 验证左侧导航栏显示"更新数据"选项
**步骤**:
1. 打开Web界面 http://127.0.0.1:5000
2. 查看左侧导航栏

**预期结果**:
- ✓ 左侧导航栏显示"更新数据"选项
- ✓ 位置在"策略配置"下方，用分隔线分开
- ✓ 图标为🔄，文字为"更新数据"

### TC-002: 验证点击"更新数据"按钮触发更新
**步骤**:
1. 打开Web界面
2. 点击左侧导航栏的"更新数据"
3. 在确认对话框中点击"确定"

**预期结果**:
- ✓ 弹出确认对话框："确定要更新数据吗？这可能需要几分钟时间。"
- ✓ 点击确定后，概览页显示"数据更新进度"卡片
- ✓ 进度条开始显示更新进度

### TC-003: 验证更新进度显示
**步骤**:
1. 触发数据更新
2. 观察进度卡片

**预期结果**:
- ✓ 显示进度条（0-100%）
- ✓ 显示成功/失败数量
- ✓ 显示实时状态消息
- ✓ 进度条平滑更新

### TC-004: 验证更新完成后的处理
**步骤**:
1. 等待更新完成
2. 观察完成后的行为

**预期结果**:
- ✓ 进度条达到100%
- ✓ 弹出完成提示："数据更新完成！成功: X 失败: Y"
- ✓ 进度卡片隐藏
- ✓ 统计信息自动刷新

### TC-005: 验证更新过程中的防护
**步骤**:
1. 触发数据更新
2. 在更新过程中再次点击"更新数据"

**预期结果**:
- ✓ 第二次点击被忽略或提示"已有更新任务在运行中"
- ✓ 不会启动第二个更新任务

### TC-006: 验证API接口
**步骤**:
1. 使用Postman或curl测试API

**测试命令**:
```bash
# 触发更新
curl -X POST http://127.0.0.1:5000/api/update \
  -H "Content-Type: application/json" \
  -d '{"max_stocks": null}'

# 获取更新状态
curl http://127.0.0.1:5000/api/update/status
```

**预期结果**:
- ✓ /api/update 返回 {"success": true, "message": "数据更新已启动", ...}
- ✓ /api/update/status 返回当前更新状态

## 测试说明

1. **进度显示**: 前端每秒轮询一次 `/api/update/status` 接口获取最新状态
2. **后台执行**: 更新任务在后台线程中执行，不阻塞Web服务
3. **状态管理**: 全局 `update_status` 字典维护更新状态
4. **错误处理**: 更新失败时显示错误信息，用户可重试
